  <files>.planning/phases/03-webhook-backend/schema-migration.sql</files>
  <action>
    Write the following SQL to `.planning/phases/03-webhook-backend/schema-migration.sql` — this file is the authoritative migration artifact to hand to the user for execution in the Supabase SQL editor.

    The SQL must do three things:
    1. Add `external_id TEXT` column (nullable, no default) to `treasury.budget_line_items` using `ADD COLUMN IF NOT EXISTS` so the statement is idempotent.
    2. Add `source TEXT DEFAULT 'csv'` column to `treasury.budget_line_items` using `ADD COLUMN IF NOT EXISTS`. The DEFAULT ensures all existing rows get source='csv' immediately.
    3. Create a UNIQUE index named `idx_line_items_external_id_source` on `(external_id, source)` with `WHERE external_id IS NOT NULL` using `CREATE UNIQUE INDEX IF NOT EXISTS` so re-running is safe.

    Exact SQL:
    ```sql
    -- Phase 3 schema migration: webhook deduplication columns
    -- Apply via: Supabase Dashboard → SQL Editor
    -- Safe to run multiple times (IF NOT EXISTS guards)

    ALTER TABLE treasury.budget_line_items
      ADD COLUMN IF NOT EXISTS external_id TEXT,
      ADD COLUMN IF NOT EXISTS source TEXT DEFAULT 'csv';

    CREATE UNIQUE INDEX IF NOT EXISTS idx_line_items_external_id_source
      ON treasury.budget_line_items (external_id, source)
      WHERE external_id IS NOT NULL;
    ```

    Also prepare a verification query to save alongside:
    ```sql
    -- Verify migration applied correctly
    SELECT column_name, data_type, column_default, is_nullable
    FROM information_schema.columns
    WHERE table_schema = 'treasury'
      AND table_name = 'budget_line_items'
      AND column_name IN ('external_id', 'source')
    ORDER BY column_name;

    SELECT indexname, indexdef
    FROM pg_indexes
    WHERE schemaname = 'treasury'
      AND tablename = 'budget_line_items'
      AND indexname = 'idx_line_items_external_id_source';
    ```

    Write both SQL blocks to the file with comments separating them.
