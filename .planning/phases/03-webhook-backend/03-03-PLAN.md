<objective>
Apply the two targeted changes to `scripts/loadEVFinances.js` that make CSV re-imports safe once webhook rows exist: tag all CSV line items with `source: 'csv'` and exclude webhook rows from the budget clear operation.

Purpose: Without these changes, a CSV re-import after go-live would delete all webhook-sourced donation rows from the database. The source column exists to distinguish origins; these two changes are what make that distinction effective. These code changes can be authored and merged independently of the schema migration because the `source` column has a DEFAULT and the `.neq()` filter is harmless against rows without a webhook source.

Output: Two modified sections in `scripts/loadEVFinances.js` (lines ~255 and ~306-316).
</objective>
