  <name>Task 3: Analyze logs, resolve ambiguities, apply fixes if needed</name>
  <files>supabase/functions/givebutter-webhook/index.ts</files>
  <action>
    Read the log values provided by the user in Task 2. Resolve both open ambiguities:

    **Ambiguity A — Signature algorithm:**
    Compare the `signatureHeader` value from the log against the GIVEBUTTER_SIGNING_SECRET value (user knows this from Plan 04 Task 2).
    - If they match character-for-character: raw string comparison is correct. No code change needed. Document as confirmed.
    - If they do NOT match: GiveButter is likely computing HMAC-SHA256(body, secret) and sending the hex digest. Implement HMAC-SHA256 verification:

    ```typescript
    // Replace the timingSafeEqual call in the signature check with:
    async function verifyHmacSha256(body: string, secret: string, receivedSig: string): Promise<boolean> {
      const enc = new TextEncoder()
      const key = await crypto.subtle.importKey(
        'raw', enc.encode(secret), { name: 'HMAC', hash: 'SHA-256' }, false, ['sign']
      )
      const sigBuffer = await crypto.subtle.sign('HMAC', key, enc.encode(body))
      const computed = Array.from(new Uint8Array(sigBuffer))
        .map(b => b.toString(16).padStart(2, '0'))
        .join('')
      return timingSafeEqual(computed, receivedSig)
    }
    ```
    Update the handler to call `await verifyHmacSha256(rawBody, SIGNING_SECRET, sig)` instead of the direct `timingSafeEqual(sig, SIGNING_SECRET)` call.

    **Ambiguity B — Amount unit:**
    Read the `rawAmount` value from the log.
    - If `rawAmount` is `1` or `1.00` → dollars. Amount is passed directly to p_amount. No code change needed. Document as confirmed.
    - If `rawAmount` is `100` → cents. The Edge Function must divide by 100 before passing to the RPC:
      Change `p_amount: amount` to `p_amount: amount / 100` in the `supabase.rpc()` call.

    Apply whichever fixes are needed (0, 1, or 2 changes). If no fixes are needed, make no changes to the file.

    If HMAC-SHA256 fix was applied: redeploy the function:
    ```bash
    supabase functions deploy givebutter-webhook --no-verify-jwt
    ```

    If cents fix was applied: redeploy the function:
    ```bash
    supabase functions deploy givebutter-webhook --no-verify-jwt
    ```

    After any redeployment, GiveButter's delivery log may allow resending the failed event — check the retry behavior with the corrected function.
  </action>
