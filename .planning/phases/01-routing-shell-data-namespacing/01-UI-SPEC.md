## Spacing Scale

Declared values (multiples of 4 only). Source: measured from App.css existing usage, normalized to 8-point scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps (badge margin, icon-label gap) |
| sm | 8px | Compact element spacing (breadcrumb gaps, small padding) |
| md | 16px | Default element spacing (tab padding, card inner gap) |
| lg | 24px | Section padding (main content padding, card group gap) |
| xl | 32px | Hero padding (hero-section inner padding ~2.5rem maps here) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (error state top padding ~4rem) |

Exceptions:
- "Back to Cities" link touch target: minimum 44px height (accessibility — WCAG 2.5.5)
- City picker card padding: 24px (lg) — matches existing `.info-card` at 1.75rem, rounded to nearest token

## Color

All values pre-populated from `src/index.css` CSS custom properties. No new colors introduced in Phase 1.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f5` (`--light-gray`) | Page background, app shell background |
| Secondary (30%) | `#ffffff` (`--white`) | Header, cards, city picker container, nav surface |
| Accent (10%) | `#00657c` (`--muted-blue`) | Active state indicators, primary amounts, accent links |
| Destructive | `#ff5740` (`--coral`) | Error messages only — no destructive actions in this phase |

Accent reserved for:
1. Active tab / dataset tab indicator
2. Dollar amount values (info cards)
3. "Back to Cities" link on hover/focus state (underline color)
4. City name link on city picker page (hover state)

**Note on error state:** The existing error state (`Unable to load data`) uses the app's default text color (`--black: #1c1c1c`), not coral. Coral is reserved for destructive confirmations. The Phase 1 enhancement is to surface a more informative error message — color remains `--black` for body, standard red (`--coral`) only if a destructive consequence is implied. In Phase 1 the error is informational only, so use `--black` text on `--white` card background.

## Copywriting Contract

| Element | Copy |
|---------|------|
| City picker page heading | "Choose a City" |
| City picker Bloomington link | "Bloomington, IN" |
| Back to cities link | "Back to Cities" |
| Empty state heading (city picker, no cities configured) | "No cities available yet" |
| Empty state body | "Check back soon — more cities are being added." |
| Data error heading | "Unable to load budget data" |
| Data error body | "The data file could not be loaded. Check the browser console for the specific file path and HTTP status." |
| Unknown city redirect | No visible copy — redirect is silent (Navigate replace, no toast) |

**Destructive actions in Phase 1:** None. No delete, remove, or irreversible user actions exist in this phase.

**Copywriting rationale:**
- "Back to Cities" (not "Back to Home") — the destination is explicitly the city list, which is more informative to the user
- Error body now includes actionable guidance ("check console for path and status") — improvement over v1.0 "Please check the console for errors" which was developer-facing
- City picker heading "Choose a City" matches the placeholder pattern in RESEARCH.md Pattern 4 exactly

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Navigate to `/` | Renders `CityPickerPage` — "Choose a City" heading + Bloomington link |
| Navigate to `/bloomington` | Renders full budget tracker (identical to v1.0 visually) |
| Navigate to `/los-angeles` or any unknown slug | Silent redirect to `/` via `<Navigate to="/" replace />` — no flash, no error toast |
| Click "Back to Cities" | `<Link to="/">` — SPA navigation, no page reload |
| Browser back from `/bloomington` | Returns to `/` (standard browser history) |
| Data load failure (404 on data file) | Error card: white card, centered, "Unable to load budget data" + guidance copy |
| Data loading in progress | Existing loading spinner/state from v1.0 — no changes |

**Focus management:** "Back to Cities" link must be keyboard-navigable (native `<Link>` satisfies this). City picker link to Bloomington must be keyboard-navigable (native `<Link>` satisfies this).
