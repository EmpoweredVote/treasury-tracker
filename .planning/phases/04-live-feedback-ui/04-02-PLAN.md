<objective>
Connect the `useAnimatedCounter` hook (built in Plan 04-01) into the two places the revenue total is visually rendered on the nonprofit financials page, and add the green-glow "settled" acknowledgment.

Purpose: This is the user-visible payoff of the phase. When a donor returns from GiveButter with a successful donation, they see the revenue number count up from the old value to the new value (~600ms) and briefly glow green (~2s) before settling. If nothing changed (e.g., they closed GiveButter without donating), nothing animates — silent.

- RESEARCH.md (finding #3) mentions `QuickFactsRow.tsx` as the target. That component exists but is NOT actually rendered anywhere in App.tsx. The real revenue total display on financials.empowered.vote lives in TWO places that ARE rendered:

  2. `DatasetTabs.tsx` line ~79 — the `text-2xl font-bold` dollar figure on the Revenue card.
- Both consume `revenueData.metadata.totalBudget` (same source value), so both must animate together from the same `useAnimatedCounter` target.

- `PlainLanguageSummary.tsx` — modified: animate the revenue `<strong>` with count-up + green glow on settle.
- `DatasetTabs.tsx` — modified: animate the revenue card total with count-up + green glow on settle.

    Step 2 — Inside the component body, just after the existing variable derivations (after line ~89 where `formatPerResident` is defined, BEFORE the `return` at line 94), add:

    ```tsx
    // Revenue count-up animation + green-glow settle.
    // Target tracks revenueData.metadata.totalBudget; hook is no-op when target is unchanged.
    const revenueTarget = revenueData?.metadata.totalBudget ?? 0;
    const [revenueGlowing, setRevenueGlowing] = useState(false);
    const glowTimerRef = useRef<number | null>(null);

    // CRITICAL (RESEARCH finding #6): onComplete MUST be wrapped in useCallback with
    // stable deps, or the useAnimatedCounter effect resets on every render.
    const handleRevenueSettled = useCallback(() => {
      setRevenueGlowing(true);
      if (glowTimerRef.current != null) window.clearTimeout(glowTimerRef.current);
      glowTimerRef.current = window.setTimeout(() => setRevenueGlowing(false), 2000);
    }, []);

    const animatedRevenue = useAnimatedCounter(revenueTarget, 600, handleRevenueSettled);

    // Cleanup pending timer on unmount (RESEARCH finding #10).
    useEffect(() => {
      return () => {
        if (glowTimerRef.current != null) window.clearTimeout(glowTimerRef.current);
      };
    }, []);
    ```

    Step 3 — Replace the existing revenue dollar render at line ~206.

    FIND:
    ```tsx
    <strong className="text-ev-gray-800">{formatAmount(revenueData.metadata.totalBudget)}</strong>
    ```

    REPLACE WITH:
    ```tsx
    <strong
      className="text-ev-gray-800 inline-block rounded-sm px-0.5"
      style={{
        transition: 'box-shadow 700ms ease-out',
        boxShadow: revenueGlowing
          ? '0 0 0 2px #22c55e, 0 0 16px 4px rgba(34, 197, 94, 0.4)'
          : 'none',
      }}
    >{formatAmount(animatedRevenue)}</strong>
    ```

    Rationale for the inline style (RESEARCH finding #5): Tailwind v4 arbitrary box-shadow does not animate reliably via `transition-shadow` because of `@property` limitations. Direct inline style with an explicit `transition` property is the reliable approach.

    Color `#22c55e` matches Tailwind's `green-500` (the standard "success" tone used elsewhere in the EV design system for positive signals).

    - Do not animate on initial mount: `useAnimatedCounter` initializes display to target so the first render shows the target directly without animation (see hook behavior from 04-01). Verify by eye when reading the hook implementation — if the hook animates from 0 on mount, fix it in the hook, not here.
