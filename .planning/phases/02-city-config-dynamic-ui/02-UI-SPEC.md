## Interaction Contract

### City Picker — Card Grid

- Renders one card per entry in `CITY_REGISTRY` (order: registry order)
- Active card: `<Link to="/{slug}">` — full card is clickable, no inner button
- Disabled card: `<div aria-disabled="true">` — no click handler, no pointer cursor
- Focus ring: browser default outline suppressed (`:focus { outline: none }`); instead border transitions to `--muted-blue` on focus
- Keyboard: Tab navigates active cards only; disabled cards are skipped (not focusable via Link)
- Mobile breakpoint: grid collapses to single column at viewport width ≤ 480px

### Dataset Tabs — Filtering

- `DatasetTabs` receives `availableDatasets: DatasetId[]` prop from `CityPage`
- Renders only tabs present in `availableDatasets` — hidden tabs produce no DOM nodes
- Mobile dropdown: same filter applied — only configured datasets appear in the dropdown list
- Active dataset guard: if current `activeDataset` is not in `availableDatasets`, reset to `availableDatasets[0]` before render

### Year Selector

- Renders years from `cityConfig.availableYears` (descending, most recent first)
- Default selected year: `cityConfig.defaultYear`
- No UI change — only the data source changes from hardcoded array to config

### Hero Section

- Background image URL sourced from `cityConfig.heroImageUrl`
- `<h1>` text sourced from `cityConfig.heroTitle`
- No layout or visual change — only the data source changes

### Config Not Found Guard

- If `getCityConfig(slug)` returns `undefined`, `CityPage` renders `<Navigate to="/" replace />`
- No error toast, no error page — silent redirect (consistent with Phase 1 unknown-slug behavior)
